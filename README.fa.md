# SNI-Spoofing

**دور زدن DPI با دستکاری هدرهای IP/TCP**

> 🇮🇷 فارسی | 🇬🇧 [English](README.md)

---

## SNI-Spoofing چیست؟

SNI-Spoofing یک ابزار برای دور زدن **سیستم‌های فیلترینگ DPI** (Deep Packet Inspection) است که توسط اپراتورهای اینترنتی و سیستم‌های سانسور (مانند سیستم‌های فیلترینگ در ایران) برای مسدود کردن دسترسی به اینترنت آزاد استفاده می‌شود.

این ابزار با دستکاری هدرهای IP و TCP در سطح پایین، با استفاده از درایور **WinDivert**، پکت‌ها را در لحظه رهگیری و تغییر می‌دهد.

تکنیک اصلی این است که یک **TLS ClientHello جعلی** با یک SNI جعلی (نام دامنه‌ای که فیلتر نشده) ارسال می‌شود تا سیستم DPI فکر کند اتصال به یک سایت مجاز است و اجازه عبور بدهد. سپس ترافیک واقعی از طریق همان اتصال آزادانه جریان پیدا می‌کند.

---

## روش کار

۱. برنامه روی یک پورت محلی گوش می‌دهد (تنظیم شده در `config.json`)
۲. اتصال ورودی دریافت و یک TLS ClientHello جعلی با SNI جعلی ساخته می‌شود
۳. WinDivert پکت‌های خروجی را رهگیری کرده و داده جعلی را با یک شماره sequence دستکاری‌شده تزریق می‌کند (روش bypass: `wrong_seq`)
۴. سیستم DPI SNI مجاز را می‌بیند و اتصال را تأیید می‌کند
۵. پس از اتمام فرآیند bypass، رله دوطرفه کامل بین کلاینت و سرور مقصد شروع می‌شود

## جزئیات فنی کلیدی

**چرا شماره sequence اشتباه کار می‌کند؟**

سیستم DPI **بازرسی سطحی یا stateless** انجام می‌دهد — اولین پکت داده را می‌خواند و فیلد SNI را بررسی می‌کند، اما شماره sequence TCP را کامل اعتبارسنجی نمی‌کند. سرور مقصد stateful است و پکت خارج از پنجره را به درستی رد می‌کند. نتیجه: DPI فریب می‌خورد، سرور تحت تأثیر قرار نمی‌گیرد.

**TLS ClientHello جعلی (`utils/packet_templates.py`)**

یک TLS 1.3 ClientHello واقعی و کامل ساخته می‌شود با:
- فیلد `random` با ۳۲ بایت تصادفی
- `session_id` با ۳۲ بایت تصادفی
- `key_share` با ۳۲ بایت تصادفی
- SNI جعلی تنظیم شده روی دامنه مجاز (مثلاً `mci.ir`)
- پدینگ برای نرمال‌سازی طول پکت (در مجموع ۵۱۹ بایت)

این پکت برای هر بازرس TLS کاملاً قانونی به نظر می‌رسد — فقط SNI جعلی است.

**محاسبه شماره sequence**

```
fake_seq = (syn_seq + 1 - len(fake_payload)) & 0xffffffff
```

این کار باعث می‌شود محدوده داده پکت جعلی *قبل از* بایت مورد انتظار بعدی قرار بگیرد و خارج از پنجره دریافت سرور باشد. سرور آن را بی‌صدا دور می‌اندازد. DPI ابتدا آن را می‌بیند و SNI را ثبت می‌کند.

**فیلتر WinDivert**

```
tcp and (
  (ip.SrcAddr == LOCAL_IP and ip.DstAddr == TARGET_IP)
  or
  (ip.SrcAddr == TARGET_IP and ip.DstAddr == LOCAL_IP)
)
```

فقط پکت‌های روی مسیر خاص local↔target رهگیری می‌شوند. تمام ترافیک دیگر بدون تغییر عبور می‌کند.

**مدل ترد**

- ترد اصلی: حلقه رویداد `asyncio` — پذیرش اتصال‌ها، مدیریت تسک‌های رله
- ترد WinDivert: ترد daemon — رهگیری و تزریق پکت‌ها به صورت همگام
- Thread Pool (32 worker): جایگزین spawning ترد جداگانه برای هر اتصال — overhead کمتر
- ارتباط: `asyncio.Event` (`t2a_event`) از ترد WinDivert به حلقه async سیگنال می‌دهد

## تنظیمات (`config.json`)

```json
{
  "LISTEN_HOST":    "0.0.0.0",
  "LISTEN_PORT":    40443,
  "FAKE_SNI":       "hcaptcha.com",
  "CONNECT_IP":     "104.19.229.21",
  "CONNECT_PORT":   443,
  "LOG_CLIENT_SNI": true,
  "LOG_FILE":       ""
}
```

| کلید | توضیح |
|---|---|
| `LISTEN_HOST` | آدرس محلی برای گوش دادن (`0.0.0.0` = همه اینترفیس‌ها) |
| `LISTEN_PORT` | پورت محلی پروکسی |
| `FAKE_SNI` | دامنه جعلی برای فریب DPI (باید فیلتر نشده باشد) |
| `CONNECT_IP` | IP سرور مقصد |
| `CONNECT_PORT` | پورت سرور مقصد (معمولاً ۴۴۳) |
| `LOG_CLIENT_SNI` | `true` — لاگ کردن SNI واقعی از TLS hello کلاینت |
| `LOG_FILE` | مسیر فایل لاگ متنی (مثل `"sni.log"`). خالی = غیرفعال |
